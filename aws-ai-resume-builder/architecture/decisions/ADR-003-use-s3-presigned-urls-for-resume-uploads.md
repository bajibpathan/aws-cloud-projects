# ADR-003: Use Amazon S3 Presigned URLs for Secure Resume Uploads

## Status

Accepted

---

## Date

2026-07-21

---

## Context

The application allows users to upload resume files that contain personally identifiable information (PII). These files must be uploaded securely while preventing unauthorized access to the Amazon S3 bucket.

Several approaches were considered for handling file uploads:

1. Upload files through a backend Lambda function.
2. Allow direct uploads to a public S3 bucket.
3. Generate temporary Amazon S3 presigned URLs that allow clients to upload directly to a private S3 bucket.

The solution needed to:

* Keep the S3 bucket private.
* Minimize AWS Lambda execution time.
* Reduce infrastructure costs.
* Scale automatically for concurrent uploads.
* Follow AWS security best practices.

---

## Decision

The application uses **Amazon S3 Presigned URLs** generated by an AWS Lambda function.

The upload workflow is:

```text
Client
    │
    ▼
Amazon API Gateway
    │
    ▼
AWS Lambda
    │
Generate Presigned URL
    │
    ▼
Client uploads directly
    │
    ▼
Private Amazon S3 Bucket
```

The Lambda function validates the upload request and returns a temporary presigned URL with a limited expiration time.

The browser uploads the resume directly to Amazon S3 without the file passing through the backend.

---

## Rationale

Using presigned URLs provides several advantages:

### Improved Security

* The Amazon S3 bucket remains private.
* Upload URLs expire automatically after a short period.
* Clients receive permission to upload only a single object.
* IAM credentials are never exposed to users.

### Better Scalability

* File uploads bypass AWS Lambda.
* Large files do not consume Lambda memory or execution time.
* Amazon S3 handles uploads directly and scales automatically.

### Lower Cost

* Lambda processes only a lightweight request to generate the upload URL.
* Upload traffic does not increase Lambda execution duration.
* Reduced compute costs compared to proxying file uploads.

### Simpler Architecture

* API Gateway and Lambda handle authentication and request validation.
* Amazon S3 is responsible for storing uploaded files.
* Each service performs a single responsibility.

---

## Alternatives Considered

### Option 1 – Upload Through Lambda

**Pros**

* Centralized request processing.
* Simple application flow.

**Cons**

* Lambda becomes a bottleneck for large uploads.
* Increased execution time and cost.
* Less scalable.

**Decision**

Rejected.

---

### Option 2 – Public S3 Bucket

**Pros**

* Simplest implementation.

**Cons**

* Exposes uploaded files.
* Does not meet security requirements.
* Increases the risk of unauthorized uploads.

**Decision**

Rejected.

---

### Option 3 – Amazon S3 Presigned URLs

**Pros**

* Private S3 bucket.
* Temporary upload permissions.
* Better scalability.
* Lower operational cost.
* AWS recommended approach.

**Cons**

* Requires an additional API call to generate the upload URL.

**Decision**

Accepted.

---

## Consequences

### Positive

* Secure upload workflow.
* Private storage of resume files.
* Highly scalable architecture.
* Reduced Lambda execution time.
* Lower operating costs.
* Alignment with AWS serverless best practices.

### Negative

* Additional Lambda function required.
* Clients must request a presigned URL before uploading.
